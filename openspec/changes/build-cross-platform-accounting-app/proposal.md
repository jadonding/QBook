## 为什么

当前仓库缺少一个可执行的跨平台记账产品方案，无法把“参考一木记账并实现核心能力覆盖”的目标继续拆解到规格和实施阶段。需要先明确范围、能力边界和平台差异，才能稳定推进实现。

## 变更内容

- 新增一个以 Flutter 为客户端技术栈的跨平台记账产品方案，覆盖 Android、iOS，并为 Web 和 Desktop 预留管理与分析能力。
- 新增核心记账能力，包括收入、支出、转账、账本、分类、账户、账单管理和多维统计。
- 新增资产与预算能力，包括资金账户、信贷账户、报销、借入借出、预算和净资产视图。
- 新增效率能力，包括智能文字记账、语音记账、截图导入、周期记账和导入导出。
- 新增协作与同步能力，包括云同步、共享账本、成员管理和冲突处理策略。

## 功能 (Capabilities)

### 新增功能
- `ledger-core`: 提供账本、账户、分类、收入、支出、转账、账单列表和账单编辑等核心记账能力。
- `analytics-and-dashboard`: 提供首页数据卡片、月度和年度统计、分类占比、趋势和日历视图。
- `asset-and-budget-management`: 提供资产账户、信贷账户、报销、借入借出、预算和净资产统计。
- `smart-entry-and-import`: 提供智能文字记账、语音记账、截图导入、周期记账和 Excel/CSV 导入导出。
- `sync-and-sharing`: 提供账号同步、共享账本、成员管理、附件同步和冲突处理能力。

### 修改功能

无。

## 影响

- 将新增 Flutter 客户端项目结构、状态管理、路由和本地数据库模型。
- 将新增 OpenSpec 规范、设计文档和实施任务清单。
- 将影响后续的数据模型设计、同步接口设计、智能录入能力和平台适配方案。
