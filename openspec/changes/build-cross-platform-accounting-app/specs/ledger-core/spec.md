## 新增需求

### 需求：系统必须支持核心账本与账单管理
系统必须允许用户创建和管理账本、账户、分类，并在同一账本下记录收入、支出和转账账单。

#### 场景：创建默认账本
- **当** 新用户首次进入应用并完成初始化
- **那么** 系统必须创建一个可立即使用的默认账本

#### 场景：记录支出账单
- **当** 用户提交一笔支出并填写金额、分类、账户和时间
- **那么** 系统必须保存该账单并在账单列表中展示

#### 场景：记录收入账单
- **当** 用户提交一笔收入并填写金额、分类、账户和时间
- **那么** 系统必须保存该账单并更新对应账户余额

#### 场景：执行转账
- **当** 用户选择转出账户、转入账户并提交转账金额
- **那么** 系统必须生成一笔转账记录并同时更新两个账户余额

### 需求：系统必须支持账单查询和维护
系统必须支持账单列表展示、账单详情查看、搜索、筛选、编辑和删除。

#### 场景：按条件筛选账单
- **当** 用户按时间、分类、账户或金额范围筛选账单
- **那么** 系统必须仅展示符合条件的账单

#### 场景：编辑账单
- **当** 用户修改已存在账单的金额、分类、账户或时间
- **那么** 系统必须更新账单内容并重新计算相关账户余额

#### 场景：删除账单
- **当** 用户删除一笔账单
- **那么** 系统必须将该账单标记为删除并回滚对应的余额影响
