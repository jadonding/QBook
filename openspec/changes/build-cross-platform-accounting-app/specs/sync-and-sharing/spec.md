## 新增需求

### 需求：系统必须支持账号同步和离线使用
系统必须允许用户在离线状态下完成核心记账操作，并在恢复网络后同步本地变更。

#### 场景：离线创建账单
- **当** 用户在无网络环境下新增账单
- **那么** 系统必须先在本地保存账单并将该变更标记为待同步

#### 场景：恢复网络后同步
- **当** 设备恢复网络连接且存在待同步变更
- **那么** 系统必须尝试将待同步数据上传并更新同步状态

### 需求：系统必须支持共享账本
系统必须允许账本拥有者邀请其他成员加入账本，并根据成员身份共享账单和统计视图。

#### 场景：邀请成员加入账本
- **当** 账本拥有者发起邀请并生成分享入口
- **那么** 系统必须允许被邀请用户加入指定账本

#### 场景：共享成员查看统计
- **当** 共享成员进入已加入账本
- **那么** 系统必须展示该账本授权范围内的账单和统计信息

### 需求：系统必须处理同步冲突和附件同步
系统必须记录同步版本，并对附件和冲突数据提供可追踪的处理结果。

#### 场景：普通字段冲突
- **当** 不同设备对同一普通实体发生并发修改
- **那么** 系统必须按既定冲突策略决定最终版本并记录同步结果

#### 场景：附件上传失败
- **当** 某笔账单的附件上传失败
- **那么** 系统必须保留本地附件引用并显示上传失败状态
