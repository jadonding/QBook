## 新增需求

### 需求：系统必须支持多类型资产账户
系统必须支持资金账户、信贷账户、报销账户、借入借出账户和理财账户，并提供总资产、负资产和净资产视图。

#### 场景：查看资产概览
- **当** 用户进入资产相关页面
- **那么** 系统必须展示总资产、负资产和净资产汇总

#### 场景：隐藏账户
- **当** 用户将某个账户标记为隐藏
- **那么** 系统必须在默认列表中隐藏该账户且不得删除历史账单

### 需求：系统必须支持预算管理
系统必须允许用户设置总预算和分类预算，并在预算接近或超过限制时提供反馈。

#### 场景：设置分类预算
- **当** 用户为某个支出分类设置月预算
- **那么** 系统必须保存该预算并在统计中关联该分类的实际支出

#### 场景：预算超支
- **当** 某个预算周期内的实际支出超过预算金额
- **那么** 系统必须将该预算标记为超支

### 需求：系统必须支持报销和借贷记录
系统必须允许用户将账单标记为报销账单，并记录借入借出及后续收款还款过程。

#### 场景：部分报销
- **当** 用户为一笔报销账单录入部分到账金额
- **那么** 系统必须保留未完成状态并记录已报销金额

#### 场景：借出后收款
- **当** 用户为一笔借出记录录入回款
- **那么** 系统必须减少未结清金额并更新该记录状态
