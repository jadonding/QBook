## 新增需求

### 需求：系统必须支持智能文字记账
系统必须允许用户输入自然语言描述，并解析出账单所需的关键字段供用户确认。

#### 场景：解析支出文本
- **当** 用户输入“昨天支付宝打车花了12元”
- **那么** 系统必须识别出支出类型、金额、时间和候选账户信息

#### 场景：用户修正解析结果
- **当** 用户修改系统识别出的分类或账户
- **那么** 系统必须保存最终账单并记录本次修正规则用于后续优化

### 需求：系统必须支持语音和截图导入作为移动端增强能力
系统必须在移动端提供语音记账和截图导入能力，并允许用户在保存前修改识别结果。

#### 场景：语音转文字记账
- **当** 移动端用户发起语音记账并完成语音输入
- **那么** 系统必须将语音结果送入智能记账流程并展示待确认账单

#### 场景：截图导入待确认账单
- **当** 移动端用户导入支付截图
- **那么** 系统必须生成可编辑的待确认账单而不是直接写入正式账本

### 需求：系统必须支持周期记账和数据导入导出
系统必须支持周期规则生成账单，并支持 Excel/CSV 导入和导出。

#### 场景：生成周期账单
- **当** 用户配置按月执行的周期记账规则且达到执行时间
- **那么** 系统必须自动生成对应账单或待确认账单

#### 场景：导入 Excel 账单
- **当** 用户选择符合模板格式的 Excel 文件进行导入
- **那么** 系统必须展示导入预览并在确认后写入账本

#### 场景：导出筛选账单
- **当** 用户在筛选结果上执行导出
- **那么** 系统必须导出当前筛选范围内的账单数据
